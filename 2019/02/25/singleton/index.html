线程安全的单例创建

发表于 2019-02-25 | 分类于 多线程 | | 阅读次数:

单例模式是平时开发中常用的模式之一，在多线程环境下有以下几种创建方式。

1、单例实例在类加载时进行创建（饿汉模式）

这种创建方式简单直接，但是当初始化操作比较复杂时容易导致类加载速度缓慢，同时，如果实例创建之后没有使用，会造成资源的浪费。

2、单例实例在第一次使用时进行创建（懒汉模式）

使用synchronized关键字修饰getInstance方法能够保证只有同一时间只能有一个线程进入调用该方法，是线程安全的，但是这种方式会极大地降低运行效率，不推荐使用。

这种方式看似是线程安全的，但是由于指令重排序的原因，上面的写法也不是线程安全的。创建一个对象需要以下3步：

  1. memory = allocate() 分配对象内存空间

  2. ctorInstance() 初始化对象

  3. instance = memory 设置instance指向刚分配的内存

如果发生指令重排序，第三步操作先于第二步操作执行

  1. memory = allocate() 分配对象内存空间
  2. instance = memory 设置instance指向刚分配的内存
  3. ctorInstance() 初始化对象

可能会出现这种情况：

线程A执行到了上面代码的第16行，线程B执行到第13行。线程A创建时先将instance指向了分配的内存但是尚未初始化，而此时线程B检测到instance分配了内存，直接返回了该instance，就会出现问题。

纠正这个问题，只需要禁止指令重排序，即使用volatile关键字对instance进行修饰。

这种单例创建模式叫双重同步锁单例模式。
