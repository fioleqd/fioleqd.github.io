阻塞队列

发表于 2019-02-27 | 分类于 多线程 | | 阅读次数:

阻塞队列常用于生产者和消费者的场景，生产者是往队列中添加元素的线程，消费者是从队列中拿元素的线程。当队列满了之后，一个线程尝试进行入队列操作时会阻塞，除非另一个线程进行了出队列操作；当队列空了之后，一个线程进行出队列操作时会阻塞，除非另一个线程进行了入队列操作。

阻塞队列一共提供了四套方法，用于插入、删除、检查操作。第一套方法：如果这三个操作不能马上执行就抛出异常；第二套方法：如果操作不能马上执行就返回特定值，一般是true或false；第三套方法：如果操作不能马上执行会阻塞；第四套方法：如果操作不能马上执行会阻塞一定时间，阻塞完成后还不能执行则返回特定值。

BlockingQueue实现类：

1、 ArrayBlockingQueue

ArrayBlockingQueue是一个有界阻塞队列，内部实现是一个数组，有界的意思是其容量是有限的，必须在初始化时指定大小，一旦指定无法改变。以FIFO方式存数据。

2、 DelayQueue

DelayQueue是一个支持延时获取元素的无界阻塞队列。队列使用PriorityQueue来实现。队列中的元素必须实现Delayed接口，在创建元素时可以指定多久才能从队列中获取当前元素。只有在延迟期满时才能从队列中提取元素。我们可以将DelayQueue运用在以下应用场景：

  • 缓存系统的设计：可以用DelayQueue保存缓存元素的有效期，使用一个线程循环查询DelayQueue，一旦能从DelayQueue中获取元素时，表示缓存有效期到了。
  • 定时任务调度。使用DelayQueue保存当天将会执行的任务和执行时间，一旦从DelayQueue中获取到任务就开始执行，从比如TimerQueue就是使用DelayQueue实现的。

3、 LinkedBlockingQueue

如果初始化时给定了大小，则LinkedBlockingQueue是有界的，不指定大小即是无边界的（使用了默认整型值）。操作和ArrayBlockingQueue一致。

4、 PriorityBlockingQueue

带优先级的阻塞队列，没有边界，有排序规则，允许添加null，所有插入队列的对象都必须实现Comparator接口，队列中的优先级通过Comparator接口的实现来定义。

5、 SynchronousQueue

SynchronousQueue是一个不存储元素的阻塞队列。每一个put操作必须等待一个take操作，否则不能继续添加元素。SynchronousQueue可以看成是一个传球手，负责把生产者线程处理的数据直接传递给消费者线程。队列本身并不存储任何元素，非常适合于传递性场景,比如在一个线程中使用的数据，传递给另外一个线程使用，SynchronousQueue的吞吐量高于LinkedBlockingQueue 和 ArrayBlockingQueue。
